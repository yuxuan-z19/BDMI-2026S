# Day3 学习总结：机器学习与 scikit-learn 入门

## 1. 本周内容概览

WW03 文件夹主要围绕机器学习基础、scikit-learn 实践和一个金融数据预测项目展开。课程从机器学习的基本概念出发，介绍了监督学习、无监督学习、强化学习等不同类型，并进一步通过 scikit-learn 展示常见算法的使用流程。

本周材料还包含圆周率计算的补充内容，用来说明计算问题、数值方法和计算机性能之间的关系。整体来看，第 3 周是从 Python 数据处理过渡到机器学习建模的重要一周。

## 2. 机器学习基础概念

机器学习的核心目标是让模型从数据中学习规律，并在新数据上完成预测或判断。课程中强调了数据集、训练集、验证集、测试集等概念，也区分了分类和回归两类常见任务。

监督学习依赖带标签数据，典型任务包括垃圾邮件过滤、股票涨跌预测和图片分类。无监督学习则更关注数据自身结构，例如聚类和降维。强化学习通过智能体与环境交互获得奖励，适合决策类问题。

## 3. scikit-learn 实践流程

scikit-learn 是本周的重点工具。它提供了统一的建模接口，通常包括以下步骤：

1. 准备数据集，构造特征 `X` 和标签 `y`。
2. 将数据划分为训练集和测试集。
3. 选择模型，例如高斯朴素贝叶斯、逻辑回归、决策树或支持向量机。
4. 使用 `fit()` 完成训练。
5. 使用 `predict()` 进行预测。
6. 使用准确率、召回率等指标评价效果。

这种统一流程降低了算法切换成本，也让我意识到机器学习实验不只是写算法本身，更重要的是建立规范的数据处理和验证流程。

## 4. 数据预处理

课程材料中专门介绍了数据预处理。特征缩放是机器学习中非常重要的一步，例如 Z-score 标准化和 Min-Max 归一化都可以让不同量纲的数据变得可比较。

一些模型对数据尺度比较敏感，如果不进行特征变换，训练效果可能会受到明显影响。因此在建模前，应该先观察数据分布、缺失值、异常值和特征范围，再选择合适的预处理方法。

## 5. 金融数据预测项目

项目要求使用机器学习方法预测股票价格涨跌方向，并至少选择三种算法进行对比。数据涉及贵州茅台、宁德时代、药明康德、科大讯飞等股票。

这个任务体现了机器学习应用的完整流程：获取历史数据、构造特征、划分训练测试集、训练多个模型、比较命中率，并思考预测正确和错误带来的收益差异。它比普通分类练习更接近真实问题，因为金融数据具有噪声大、波动强、规律不稳定等特点。

## 6. 学习体会

第 3 周让我对机器学习有了更系统的认识。以前可能只关注某个算法如何调用，现在更清楚一个完整任务需要包含数据、模型、训练、评估和结果解释。scikit-learn 的优势在于接口简洁，适合快速验证想法。

后续学习中，我需要加强两点：一是理解不同算法适合的场景，二是重视数据预处理和评价指标，避免只看表面的准确率。
