# Study Memo - 第 9 周课程小结

**课程日期**: 2026-04-21
**课程主题**: 深度学习回顾、LLM API、Mamba 与视觉实践

---

## 1. 课程内容概览

本周课程一方面回顾了深度学习、RNN、Transformer 和计算机视觉中的重要内容，另一方面进一步扩展到大模型 API 调用、状态空间模型和语音情感识别等更实际的应用方向。

### 1.1 深度学习与 Transformer 回顾
* **训练流程**: 深度学习训练仍然围绕数据预处理、模型结构、损失函数、反向传播和优化器更新展开。
* **过拟合处理**: L2 正则化、Dropout、数据增强和集成学习都是常见方法。
* **Transformer**: 重点复习了 Self-Attention、Multi-Head Attention、Position Encoding 和 Mask。相比 RNN，Transformer 更适合并行训练，也更容易捕捉长距离依赖。

### 1.2 RNN 扩展与记忆网络
* **LSTM / GRU**: 通过门控机制改进普通 RNN 的长期记忆能力。
* **深度 RNN 与双向 RNN**: 一个增强层次表达，一个利用双向上下文。
* **NTM / DNC**: 通过外部记忆结构扩展神经网络的信息存储能力，这让我看到“神经网络 + 可微存储”的另一条思路。

### 1.3 LLM API 调用
* **基本流程**: 申请 `API_KEY`，确认 `BASE_URL` 和模型 ID，然后通过请求接口发送消息并解析模型输出。
* **消息角色**: `system`、`user`、`assistant` 分别对应控制指令、用户任务和历史响应。
* **上下文管理**: 当上下文过长时，需要通过上下文工程、记忆压缩或检索相关信息等方式保留关键信息。
* **同步与异步调用**: 同步调用逻辑清晰，异步调用更适合多个请求并发处理。

### 1.4 Mamba 与状态空间模型
* **SSM**: 状态空间模型用隐藏状态描述序列信息，在理论上兼顾递归推理和卷积式并行训练。
* **S4 与 Mamba**: 普通 SSM 的参数不随输入变化，而 Mamba 引入 selective mechanism，使模型能根据当前 token 调整状态更新。
* **Selective Scan**: 通过并行 scan 操作优化长序列计算，试图在 Transformer 的表达能力和 RNN 的线性复杂度之间取得平衡。

### 1.5 视觉与语音情感识别实践
* **Torchvision**: 提供数据集、图像变换、预训练模型和 NMS、RoIAlign 等视觉算子，可以快速搭建检测任务。
* **语音情感识别**: 将语音转为 Mel Spectrogram 后，可以把时频谱图当作图像输入 ViT 或 Swin Transformer，用于情绪分类。

## 2. 个人感想

这周的内容比较像是从“模型原理”过渡到“工程调用和新架构探索”。

1. **API 调用让我看到大模型应用的工程侧**：真正使用 LLM 时，不只是写 prompt，还要管理上下文、角色消息、并发请求和工具调用流程。
2. **Mamba 提供了另一种看待序列模型的方式**：它不是简单替代 Transformer，而是在长序列效率问题上给出了新的结构设计思路。
3. **视觉和语音任务之间可以迁移方法**：把语音转成时频谱图后再使用视觉模型，这种跨模态处理方式让我对“表示形式”的重要性有了更直观的理解。
