# Study Memo - 第 5 周课程小结

**课程主题**: 多层神经网络、自动微分与 PyTorch 模块化开发

---

## 1. 课程内容概览

本周课程继续推进到深度学习的基础框架，重点学习了多层神经网络、自动微分以及 PyTorch 中如何组织模型、参数和数据预处理流程。

### 1.1 多层神经网络
* **多层感知器（MLP）**: 学习了输入层、隐藏层、输出层的基本结构，以及为什么多层网络可以处理更复杂的非线性关系。
* **Softmax 与 logits**: 多分类任务中，模型先输出 logits，再通过 Softmax 转成概率分布。
* **损失函数与 SGD**: 训练过程的核心仍然是损失函数最小化和梯度下降更新。

### 1.2 自动微分（Autograd / AutoDiff）
* **数值微分、符号微分、自动微分**: 这三种方式的对比帮助我更清楚地理解了深度学习为什么依赖自动微分。
* **前向模式与反向模式**: 其中反向模式和神经网络的反向传播联系最紧密。
* **`retain_graph` 与 `create_graph`**: 这些参数让我意识到，PyTorch 不只是“自动算梯度”，还支持更细粒度的求导控制。

### 1.3 PyTorch 模块化开发
* **`nn.Module`**: 通过继承 `nn.Module` 可以把层、参数和前向传播统一到一个标准结构里。
* **参数与状态**: 学习了 `nn.Parameter`、`register_parameter` 和 `register_buffer` 的区别，也理解了 `state_dict` 在模型保存与加载中的作用。
* **容器类**: `nn.Sequential`、`ModuleList`、`ModuleDict` 等容器让模型搭建更灵活。

### 1.4 数据预处理
* **`transform` 与 `target_transform`**: 前者主要处理特征，后者主要处理标签。
* **`ToTensor` 与 `Lambda`**: `ToTensor` 负责把图像数据转成 Tensor，`Lambda` 则可以方便地做独热编码等自定义转换。

## 2. 个人感想

这周学习下来，我对“深度学习模型”这件事的理解更完整了，不再只是停留在“搭一个网络”的层面。

1. **我对自动微分的认识更具体了**：以前只是知道 PyTorch 能自动求导，这次把计算图、链式法则和高阶求导放在一起看后，才真正理解它为什么能高效工作。
2. **模块化写法更像工程开发**：`nn.Module` 和各种容器类让我感觉，神经网络不是零散堆出来的，而是可以像软件组件一样组织和复用。
3. **数据处理和模型本身同样重要**：`transform`、`target_transform` 这些接口虽然看起来简单，但它们实际上决定了数据进入模型前的格式，这一步不能忽略。
