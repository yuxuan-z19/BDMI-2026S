## 1) Multi-layer Neural Networks

### Key points

- A multi-layer neural network has an **input layer**, one or more **hidden layers**, and an **output layer**.
- A **fully connected / dense layer** connects every neuron in one layer to every neuron in the next layer.
- An **MLP** is a common fully connected feedforward network.
- Network connection patterns:
  - **Feedforward network**: information flows forward only.
  - **Feedback / recurrent network**: information can loop back, useful for sequential data.
  - **Memory network**: uses stored past information.

### Simple PyTorch MLP

```python
import torch
import torch.nn as nn

class SimpleMLP(nn.Module):
    def __init__(self, in_dim=784, hidden_dim=128, out_dim=10):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(in_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, out_dim)
        )

    def forward(self, x):
        return self.net(x)

model = SimpleMLP()
x = torch.randn(32, 784)
logits = model(x)
print(logits.shape)  # torch.Size([32, 10])
```

## 3) Convolutional Neural Networks (CNNs)

### Key points

CNNs are mainly used for image and visual data. Instead of connecting every pixel to every neuron, CNNs use small learnable filters that slide across the image.

Important terms:

- **Convolution**: applying a filter over local regions of an input.
- **Kernel / Filter**: learnable matrix used to detect patterns.
- **Padding**: adding border pixels, often zeros.
- **Stride**: how far the filter moves each step.
- **Channel**: input depth, such as RGB channels.
- **Feature map**: output after applying a filter.

CNNs are useful because they learn local patterns such as edges, textures, shapes, and object parts.

---

## 4) Convolution Output Size

For a 2D convolution:

```text
Output size = floor((Input size + 2 * Padding - Kernel size) / Stride) + 1
```

Example:

```text
Input size  = 32
Kernel size = 3
Padding     = 1
Stride      = 1

Output size = floor((32 + 2*1 - 3) / 1) + 1
            = 32
```

So the output remains `32 x 32`.

### PyTorch example

```python
import torch
import torch.nn as nn

conv = nn.Conv2d(
    in_channels=3,
    out_channels=16,
    kernel_size=3,
    stride=1,
    padding=1
)

x = torch.randn(8, 3, 32, 32)
out = conv(x)
print(out.shape)  # torch.Size([8, 16, 32, 32])
```

## 5) Pooling, Normalization, and Dropout

### Pooling

Pooling reduces the spatial size of feature maps.

- **Max pooling**: keeps the largest value in each local window.
- **Average pooling**: takes the average value.

Pooling reduces computation and makes the model less sensitive to small shifts.

### Normalization

Batch Normalization helps stabilize training by normalizing activations in a mini-batch.

### Dropout

Dropout randomly disables some neurons during training to reduce overfitting.

### CNN block example

```python
import torch.nn as nn

cnn_block = nn.Sequential(
    nn.Conv2d(3, 16, kernel_size=3, padding=1),
    nn.BatchNorm2d(16),
    nn.ReLU(),
    nn.MaxPool2d(kernel_size=2),
    nn.Dropout(p=0.25)
)
```

## 6) Simple CNN Model

```python
import torch
import torch.nn as nn

class SimpleCNN(nn.Module):
    def __init__(self, num_classes=10):
        super().__init__()
        self.features = nn.Sequential(
            nn.Conv2d(1, 16, kernel_size=3, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2),
            nn.Conv2d(16, 32, kernel_size=3, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2)
        )

        self.classifier = nn.Sequential(
            nn.Flatten(),
            nn.Linear(32 * 7 * 7, 128),
            nn.ReLU(),
            nn.Linear(128, num_classes)
        )

    def forward(self, x):
        x = self.features(x)
        return self.classifier(x)

model = SimpleCNN()
x = torch.randn(32, 1, 28, 28)
logits = model(x)
print(logits.shape)  # torch.Size([32, 10])
```

## 8) Transformer Overview

### Key points

A Transformer is an **encoder-decoder** architecture originally used for sequence-to-sequence tasks like machine translation.

Main components:

- Self-attention
- Scaled dot-product attention
- Positional encoding
- Masking
- Multi-head attention
- Feedforward layers

Basic flow:

```text
Input sequence -> Encoder -> Context representation -> Decoder -> Output sequence
```

---

## 9) Self-Attention

Self-attention lets each token decide which other tokens are important.

For each token, the model creates:

- **Q / Query**: what the token is looking for
- **K / Key**: what each token offers for matching
- **V / Value**: information passed forward

Formula:

```text
Attention(Q, K, V) = softmax((QK^T) / sqrt(d_k)) V
```

Meaning:

- `QK^T` calculates similarity scores.
- `sqrt(d_k)` prevents scores from becoming too large.
- `softmax` converts scores into attention weights.
- The output is a weighted combination of values.

---

## 10) Positional Encoding, Masking, and Multi-head Attention

### Positional encoding

Transformers do not process tokens in order like RNNs, so positional encoding is added to represent token order.

### Masking

Masking controls what tokens can be attended to.

Common uses:

- Ignore padding tokens.
- Stop decoder-only models from seeing future tokens.

### Multi-head attention

Multi-head attention runs several attention heads in parallel so the model can learn different relationships at the same time.

## 11) ViT, BERT, GPT, and T5

### Vision Transformer (ViT)

ViT treats image patches like tokens:

```text
Image -> Patches -> Patch embeddings -> Positional embeddings -> Transformer encoder -> Classifier
```

### Transformer model families

| Model | Structure | Main use |
|---|---|---|
| BERT | Encoder-only | Understanding / classification |
| GPT | Decoder-only | Text generation |
| T5 | Encoder-decoder | Text-to-text tasks |

## 12) PyTorch Training Flow

Training usually follows this loop:

```text
Forward pass -> Calculate loss -> Backpropagation -> Optimizer update -> Repeat
```

### Hyperparameters

- **Epochs**: number of times the model sees the full dataset.
- **Batch size**: number of samples before one parameter update.
- **Learning rate**: step size for parameter updates.

Example:

```python
learning_rate = 1e-3
batch_size = 64
epochs = 5
```

## 13) Loss Functions and Optimizers

### Loss functions

- `nn.MSELoss()` for regression
- `nn.CrossEntropyLoss()` for multi-class classification
- `nn.NLLLoss()` for log-probability outputs

### Optimizers

- **SGD**: simple baseline optimizer
- **Adam**: adaptive learning rate, commonly used in deep learning
- **RMSprop**: often used for recurrent networks
- **Adagrad**: adapts learning rate per parameter

```python
import torch.nn as nn
import torch.optim as optim

loss_fn = nn.CrossEntropyLoss()

```

## 14) PyTorch Optimization Steps

The three key lines are:

```python
optimizer.zero_grad()
loss.backward()
optimizer.step()
```

Meaning:

1. `optimizer.zero_grad()` clears old gradients.
2. `loss.backward()` computes gradients.
3. `optimizer.step()` updates model parameters.

### Training loop

```python
def train_loop(dataloader, model, loss_fn, optimizer):
    size = len(dataloader.dataset)
    model.train()

    for batch, (X, y) in enumerate(dataloader):
        pred = model(X)
        loss = loss_fn(pred, y)

        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

        if batch % 100 == 0:
            current = batch * len(X)
            print(f"loss: {loss.item():>7f}  [{current:>5d}/{size:>5d}]")
```
