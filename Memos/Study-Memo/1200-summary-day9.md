# Day9 学习总结：RNN 扩展、视觉模型、多模态与时频谱阅读

## 1. 本周内容概览

WW09 文件夹包含深度学习概述、RNN 变体、DNC 扩展、计算机视觉课件、多模态视觉大模型、Mamba 资料、LLM API 调用、目标检测以及时频谱图补充阅读。

这一周的内容比较综合：既回顾深度学习和视觉模型，又扩展到序列建模的新结构，并通过时频谱图材料连接到语音和音频分析。

## 2. 深度学习概述回顾

深度学习概述部分重新梳理了线性分类器、损失函数、深层人工神经网络和训练过程。通过回顾这些基础，可以把前几周学到的逻辑回归、神经元、CNN、Transformer 联系起来。

深度学习模型的共同核心是通过可学习参数拟合数据规律。不同网络结构的差异，本质上是对数据结构做了不同假设，例如图像的局部性、文本的序列性和音频的时频结构。

## 3. RNN 变体与 DNC

RNN 适合序列数据，但普通 RNN 在长序列上容易丢失早期信息。LSTM 和 GRU 通过门控机制缓解梯度消失，使模型能够保留更长期的上下文。

DNC 等扩展结构进一步引入外部记忆，希望模型能读写更复杂的信息。这类方法体现了一个思路：当模型内部隐藏状态不足以表达长期依赖时，可以设计显式记忆机制。

## 4. Mamba 与序列建模新方向

WW09 中包含 Mamba 相关资料。Mamba 属于近年来序列建模的新方向，试图在长序列处理上获得更好的效率。相比标准 Transformer 的注意力机制，状态空间模型类方法更关注线性复杂度和长上下文扩展能力。

这说明序列建模仍在快速发展。Transformer 虽然强大，但在超长序列和推理成本方面仍有问题，因此新的结构仍有研究价值。

## 5. 计算机视觉与多模态

本周继续包含 CV 和多模态视觉大模型内容。视觉模型负责从图像中提取语义，多模态模型则进一步把视觉信息与语言信息对齐。

在实际应用中，多模态能力可以支持图片问答、图像描述、视觉推理和智能标注。这类系统通常不仅要求识别物体，还要理解场景关系和用户问题。

## 6. LLM API 调用

LLM API 调用材料体现了大模型应用开发的基本方式。通过 API，可以把语言模型集成到自己的程序中，实现问答、摘要、代码辅助、数据分析等功能。

使用 API 时需要注意 prompt 设计、返回结果解析、错误处理、成本控制和隐私安全。模型调用不是简单发送一句话，而是需要围绕任务构建稳定流程。

## 7. 时频谱图补充阅读

时频谱图材料与语音情感识别相关。语音信号本身是一维时间序列，但通过短时傅里叶变换或梅尔频谱等方法，可以转换成二维时频表示。

这样一来，音频问题就能借鉴计算机视觉方法，例如使用 CNN 或 Vision Transformer 识别频谱图中的模式。补充论文中的 Swin Transformer 思路说明，视觉模型也可以迁移到语音情感识别等任务。

## 8. 学习体会

第 9 周给我的感觉是知识开始交叉融合：RNN、Transformer、Mamba 都在处理序列；CV、多模态和时频谱图又把图像方法扩展到更多数据类型。

后续学习中，我需要建立“数据形式决定模型结构”的意识。不同任务不是孤立的，音频可以转成频谱图，文本可以转成 token 序列，图像可以切成 patch，关键在于找到合适的表示方式。
