# Week 10 Study Memo — Automatic Speech Recognition (ASR)

## 1. Human Auditory Perception

- The human auditory system processes sound through: **Pinna → Eardrum → Ossicles (malleus, incus, stapes) → Cochlea → Auditory nerve → Brain**
- Analogous to the visual pathway, the auditory pathway provides inspiration for speech recognition systems.

---

## 2. Speech Recognition Fundamentals

### What is ASR?
- **Automatic Speech Recognition (ASR)**: Using computer programs to convert speech signals into text.
- **LVCSR**: Large Vocabulary Continuous Speech Recognition — the ultimate goal.

### Challenges in ASR
| Category | Examples |
|----------|---------|
| **Pronunciation** | Accents, tone, emotion, speaking style |
| **Linguistic** | Vocabulary size, homophones |
| **Environmental** | Noise, recording quality, cocktail party problem |

### Speech Signal Representations
| Representation | Description |
|---------------|-------------|
| **Waveform** | Time-domain signal |
| **Frequency (FFT)** | Frequency-domain via Fast Fourier Transform |
| **Spectrogram (STFFT)** | Time-frequency representation (most useful for ASR) |

---

## 3. Evaluation Metric — Word Error Rate (WER)

$$\text{WER} = \frac{\text{Substitutions} + \text{Insertions} + \text{Deletions}}{\text{Total words in reference}}$$

### Example
- Reference: "my name is Andy Chen"
- Predicted: "my nick name are Andy Chen"
- Errors: 1 insertion + 1 substitution + 1 deletion = 3
- WER = 3/5 = **60%**

> Note: WER can exceed 100% (e.g., reference "recognize speech" → predicted "wreck a nice beach" → WER = 200%)

## 4. Traditional ASR Methods

### Classic Pipeline
1. **STFT**: Convert audio to short-term vector sequences
2. **MFCC**: Extract mel-frequency cepstral coefficients
3. **Acoustic model**: Map features to phonemes
4. **Language model**: Map phonemes to words

### Classic Techniques
- **HMM** (Hidden Markov Model)
- **GMM** (Gaussian Mixture Model)
- **Viterbi search** algorithm
- **N-gram** language models

---

## 5. Deep Learning for ASR

### Evolution
- Pre-2006: Traditional methods dominated
- Post-2006: DNN integration reduced WER by **30%+**, breaking the usability barrier

### Hybrid Approach ("Alloy Model")
- Replace individual classic components with DNNs while keeping the overall pipeline.
- DNN + HMM, context-dependent models, Viterbi search, n-gram LM
- Drawback: Complex training, heavy domain expertise needed.

### End-to-End Deep Learning ASR
- **Three components**: Feature extraction → Acoustic model (NN) → Language model (NN)
- Input: Raw audio (waveform/spectrogram)
- Output: Text transcription

### CTC (Connectionist Temporal Classification)
- Proposed by **Alex Graves** (DeepMind) at ICML 2006.
- Solves the **alignment problem**: input speech X and output text Y have different lengths with no predetermined alignment.
- **CTC is alignment-free**: Considers all possible alignments and computes probabilities.
- **CTC Collapsing**: Merges repeated characters and removes blank tokens.

### Key Models
| Model | Innovation |
|-------|-----------|
| **DeepSpeech2** | End-to-end RNN-based ASR (Baidu) |
| **CLDNN** | Combines CNN + LSTM + DNN (Google) |
| **Whisper** | Large-scale ASR by OpenAI using Transformer |

### Google CLDNN Architecture
1. Time-domain convolution
2. Frequency-domain convolution (reduce spectral variation)
3. Three LSTM layers
4. One DNN (fully connected) output layer

---

## 6. The Essence of Speech Recognition

- Mathematically: $w = F(v, h)$ where $v$ is speech signal, $w$ is text output, $h$ is hidden state.
- Deep learning approach: Use deep neural networks to **approximate the function F**.
- With enough training data and compute, DNN-based LVCSR systems achieve state-of-the-art performance.
