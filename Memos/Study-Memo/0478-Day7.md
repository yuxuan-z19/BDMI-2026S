# Study Memo - 第 7 周课程小结

**课程主题**: NLP、Word2Vec、Transformer 与 Hugging Face

---

## 1. 课程内容概览

本周课程从自然语言处理的基础任务讲起，逐步过渡到词向量、Attention、预训练模型和 Hugging Face 的实际使用。

### 1.1 NLP 与语言模型
* **NLP 任务**: 包括语法检查、情感分析、语义相似性判断等。
* **语言模型**: 核心目标是根据前文预测下一个词，评价指标包括 `Cross-Entropy` 和 `Perplexity`。
* **N-gram 到 LLM**: 课上梳理了统计语言模型、神经网络语言模型到大模型的发展路线。

### 1.2 Word Embedding 与 Word2Vec
* **词向量**: 把 One-Hot 的稀疏表示映射到稠密空间，让词语之间有了可计算的语义关系。
* **CBOW / Skip-Gram**: 分别是用上下文预测中心词、用中心词预测上下文词。
* **Negative Sampling / NCE**: 用采样近似 full softmax，减少训练开销。

### 1.3 Attention 与 Transformer
* **Attention**: 通过权重分配聚合上下文信息。
* **Self-Attention / Multi-Head Attention**: Q、K、V 的计算方式让我更清楚地理解了 Transformer 如何建模长距离依赖。
* **Masked Self-Attention**: 用于自回归生成，避免模型看到未来信息。
* **BERT / T5 / BART / GPT**: 课上也对几类经典预训练模型做了对比，帮助我理解 encoder-only、decoder-only 和 encoder-decoder 的区别。

### 1.4 Hugging Face 与推理效率
* **Hugging Face**: 通过 `pipeline`、`AutoTokenizer` 和 `AutoModel` 可以快速完成文本分类、填空、生成等任务。
* **KV Cache**: 在逐 token 生成时缓存历史 Key/Value，减少重复计算。
* **MHA / MQA / GQA / MLA**: 这些注意力变体说明，大模型不仅要追求效果，也要兼顾推理速度和显存占用。

## 2. 个人感想

这周让我感受到，NLP 不是单纯“处理文字”，而是要同时解决表示、建模和效率三个层面的问题。

1. **词表示是基础**：Word2Vec 让我更直观地理解了词向量的意义，也看到了语义相近词在向量空间里的关联。
2. **Attention 是核心转折点**：相比早期按顺序处理上下文的方法，Attention 更适合捕捉全局依赖，这也是 Transformer 能成为主流的重要原因。
3. **工具链很实用**：Hugging Face 的接口让模型调用变得很直接，也让我更容易把理论知识和实际推理流程对应起来。
