## 1) Deep Learning Recap

### What I learnt

Deep learning builds on supervised learning:

- A **linear classifier** maps features to class scores.
- **SVM** tries to find a separating boundary with a large margin.
- A **loss function** measures prediction error.
- **Optimization** updates model parameters to reduce loss.
- **Backpropagation** computes gradients through a computation graph.

Common neural network types:

- **MLP / Fully Connected Network**: dense layers for general prediction.
- **CNN**: useful for images and spectrogram-like inputs.
- **RNN / LSTM**: useful for sequence data such as speech.
- **Transformer**: attention-based architecture for sequence modelling.

## 2) Automatic Speech Recognition (ASR)

### What I learnt

**ASR** means using a computer program to convert speech into text.

Common ASR challenges:

- Different accents, speaking styles, emotions, and tones
- Background noise and poor recording quality
- Homophones and large vocabulary size
- Multiple speakers talking at the same time, also called the **cocktail party problem**

Speech signal representations:

- **Waveform**: raw audio amplitude over time.
- **Frequency spectrum**: frequency-domain view using FFT.
- **Spectrogram**: time-frequency representation, usually more useful for deep learning.

---

## 3) Evaluation: Word Error Rate (WER)

### What I learnt

**WER** is the main metric for ASR.

```text
WER = (Substitutions + Insertions + Deletions) / Number of words in reference
```

Example:

```text
Reference:  my name is Andy Chen
Prediction: my nick name are Andy Chen
```

There are insertion, substitution, and deletion errors.

Important notes:

- WER depends on the best alignment between reference and prediction.
- WER can be greater than 100% if there are many insertions.

## 4) Traditional ASR Pipeline

### What I learnt

Traditional ASR separates the problem into several components.

Basic idea:

```text
Speech signal → acoustic features → phonemes → words → final transcript
```

Typical components:

- **Acoustic Model**: estimates how likely the audio matches sounds / phonemes.
- **Language Model**: estimates how likely a word sequence is.
- **HMM / GMM**: traditional statistical modelling methods.
- **Viterbi search**: finds the most likely sequence.
- **n-gram LM**: estimates word sequence probability using nearby words.

Feature extraction:

- Audio is split into short frames.
- Features such as **MFCC** or spectrogram features are extracted.
- These features are passed into the recognition model.

## 5) Deep Learning for ASR

### What I learnt

Deep learning ASR reduces the need for manually designed pipeline stages.

Important ideas:

- **RNN / LSTM** can model temporal dependencies in speech.
- **LSTM** helps reduce vanishing/exploding gradient issues in long sequences.
- **CTC** allows training without frame-level alignment between audio and text.
- **End-to-end ASR** maps audio features directly to text or token sequences.

Why LSTM helps:

- Speech is sequential.
- The meaning of a sound frame depends on surrounding frames.
- LSTM can store useful context through gates and memory cells.

## 6) Whisper

### What I learnt

**Whisper** is an ASR system based on a Transformer Seq2Seq model.

Pipeline:

```text
Audio → 30s chunks → log-Mel spectrogram → encoder → decoder → text
```

Key points:

- Trained on large-scale multilingual and multitask supervised data.
- Uses special tokens to support tasks such as:
  - language identification
  - speech transcription
  - translation
  - timestamps
- Uses a Transformer encoder-decoder structure.
- Works well because it learns from diverse audio conditions and languages.

## 7) PyTorch Audio Recognition

### What I learnt

Audio classification can be built using PyTorch and torchaudio.

Typical preprocessing steps:

1. Load audio.
2. Resample to a fixed sampling rate.
3. Convert stereo to mono if needed.
4. Pad or trim to a fixed duration.
5. Convert waveform to Mel spectrogram.
6. Normalize the spectrogram.
7. Feed into a CNN classifier.

A simple model can use:

- `Conv2d`
- `ReLU`
- `MaxPool2d`
- `Flatten`
- `Linear`
- `Dropout`

Example workflow:

```python
import torch
import torch.nn as nn
import torchaudio.transforms as T

mel = T.MelSpectrogram(
    sample_rate=16000,
    n_fft=1024,
    hop_length=512,
    n_mels=64
)

criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
```

## 9) Memory-Efficient Sequence Models

### What I learnt

Long sequences create memory and compute problems.

Related models / methods:

- **Memformer**
  - Adds external dynamic memory to Transformer.
  - Aims for linear time and constant memory space for long sequences.
  - Uses memory reading and writing.

- **SnapKV**
  - Compresses the KV cache for long-context LLM inference.
  - Uses attention patterns before generation to identify important KV positions.
  - Helps reduce memory usage and decoding cost.

Main idea:

Instead of keeping everything, efficient models try to keep the useful context while reducing memory and computation.
