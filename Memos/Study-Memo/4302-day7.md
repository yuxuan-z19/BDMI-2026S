# Week 7 Study Memo — RNN, LSTM, Transformer & NLP

## 1. Recurrent Neural Networks (RNN)

### Motivation
- Feedforward networks (MLP, CNN) process fixed-size inputs.
- RNNs handle **sequential data** (text, speech, time series) by maintaining a hidden state across time steps.
- Applications: poetry composition (Jiuge AI system), couplet writing, machine translation.

### Vanilla RNN Structure
- **Feedback connections**: Current hidden state feeds into the next time step.
- **Weight sharing**: Same weight matrices U, V, W are used at every time step.

| Matrix | Connection |
|--------|-----------|
| **U** | Input → Hidden |
| **V** | Hidden → Output |
| **W** | Hidden → Hidden (recurrent) |

### RNN Equations
- Hidden state: $h_t = f(U \cdot x_t + W \cdot h_{t-1} + b)$
- Output: $o_t = V \cdot h_t + c$
- Training: **Back-Propagation Through Time (BPTT)**

### RNN Problems
- **Vanishing gradients**: Gradients shrink exponentially over long sequences
- **Exploding gradients**: Gradients grow uncontrollably → solved with **gradient clipping**

## 2. LSTM (Long Short-Term Memory)

### Purpose
- Designed to capture **long-range dependencies** that vanilla RNNs struggle with.
- Uses **gate mechanisms** to control information flow.

### Gate Structure
| Gate | Function |
|------|----------|
| **Forget gate** | Decides what information to discard from cell state |
| **Input gate** | Decides what new information to store |
| **Output gate** | Decides what to output based on cell state |

### GRU (Gated Recurrent Unit)
- Simplified version of LSTM with only 2 gates (reset & update).
- Fewer parameters, comparable performance.

## 3. Transformer Architecture (Deep Dive)

### Background
- 2018: Called the "ImageNet moment" for NLP — large models (ELMo, ULMFiT) on massive datasets.
- OpenAI's breakthrough: GPT (Generative Pre-trained Transformer).

### Core Innovation
Transformer replaces sequential processing (RNN) with **parallel self-attention**.

### Attention vs. Self-Attention vs. Multi-Head Attention
| Concept | Description |
|---------|-------------|
| **Attention** | General mechanism — Q and K/V can come from different sources (e.g., encoder→decoder) |
| **Self-Attention** | Special case — Q, K, V all come from the **same** input sequence |
| **Multi-Head Attention** | Multiple self-attention heads in parallel, each with independent Q/K/V weights |

### Self-Attention Mechanism
- Relates different positions within a single sequence to compute contextualized representations.
- Example: In "Time flies like arrow" vs. "Fruit flies like banana," the word "flies" has different meanings — self-attention captures this.
- Self-attention can capture syntactic features (sentence structure) and semantic features (word meaning) within a sentence.

### Attention Computation — 3 Steps
1. **Step 1 — Compute similarity**: Calculate $F(Q, K_i)$ between the Query and each Key using dot product, yielding raw relevance scores $s_i$
2. **Step 2 — Normalize**: Apply Softmax to obtain attention weights $a_i = \text{softmax}(s_i)$
3. **Step 3 — Weighted sum**: Compute output as weighted sum of Values: $\text{Attention} = \sum a_i \cdot V_i$

### Scaled Dot-Product Attention
$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$

Where:
- **Q** (Query), **K** (Key), **V** (Value) are projections of the input
- $d_k$ is the dimension of keys (dividing by $\sqrt{d_k}$ standardizes the output and prevents large dot products)

### Multi-Head Attention (MHA)
- Run multiple attention heads in parallel, each with independent Q/K/V weight matrices.
- Each head captures **different feature dimensions** (e.g., syntax, semantics, position).
- Concatenate all head outputs and project through a final linear layer.
- Complexity: $O(h \cdot n^2)$ where $h$ = number of heads, but per-head dimension is reduced.

### Masked Self-Attention (Decoder)
- Used in the **decoder** to ensure autoregressive generation (no "peeking" at future tokens).
- Applies a **causal mask** (lower-triangular matrix) setting future positions to $-\infty$ before Softmax.
- Guarantees that generating token $t$ only depends on tokens $1, 2, \ldots, t-1$.

| Feature | Self-Attention (Encoder) | Masked Self-Attention (Decoder) |
|---------|------------------------|---------------------------------|
| Visibility | Full sequence (bidirectional) | Current + past only (unidirectional) |
| Mask | Optional padding mask | **Causal mask** + padding mask |
| Used by | BERT, ViT | GPT, Transformer decoder |

### Attention Architecture Evolution
| Variant | Innovation |
|---------|------------|
| MHA | Standard multi-head attention |
| MQA (Multi-Query Attention) | Shared K/V across heads, lower compute |
| GQA (Grouped Query Attention) | Compromise between MHA and MQA |
| MLA (Multi-head Linear Attention) | Efficient for long sequences |

### Positional Encoding
- Since Transformer has no inherent notion of order, **positional encodings** are added to embeddings.
- Uses sine/cosine functions at different frequencies.

### Transformer Components
| Component | Encoder | Decoder |
|-----------|---------|---------|
| Self-attention | ✓ | ✓ (masked) |
| Cross-attention | — | ✓ |
| Feed-forward | ✓ | ✓ |
| Add & Norm | ✓ | ✓ |

### Why Transformer Wins
- **Parallel computation**: Unlike RNN, all positions processed simultaneously
- **Long-range dependencies**: Attention directly connects any two positions
- **Scalable**: Stack more encoder/decoder layers for larger models

## 4. NLP & Word Embeddings

### Word2Vec
- Maps words to dense vectors where semantically similar words are close.
- Famous example: `King - Man + Woman ≈ Queen`

### Transformer-based NLP Models
| Model | Architecture | Specialization |
|-------|-------------|----------------|
| BERT | Encoder-only | Language understanding |
| GPT | Decoder-only | Language generation |
| T5 | Encoder-decoder | Text-to-text tasks |
| BART | Encoder-decoder | Denoising autoencoder |

### HuggingFace Transformers
- Open-source library providing pre-trained models for NLP tasks.
- Standardized API for fine-tuning on downstream tasks.

---

## 5. Efficient Transformer Inference

### LLM Inference Phases
1. **Prompt phase**: Process input to build KV cache
2. **Token generation**: Use KV cache to generate tokens incrementally

### Acceleration Techniques
- **Sparse attention**: Reformer (locality-sensitive hashing), Sparse Transformer
- **KV cache optimization**: Reduce memory footprint
- **Semantic prompt caching**: Cache responses for similar prompts (up to 100× speedup)
