## 1) Deep Learning Recap

### What I learnt

Deep learning builds from simple supervised learning ideas:

- **Linear classifier**: maps input features to class scores.
- **SVM**: tries to separate classes using a maximum-margin boundary.
- **Loss function**: measures how wrong the prediction is.
- **Optimization**: updates parameters to reduce loss.
- **Backpropagation**: computes gradients through a computation graph.

Common deep learning models:

- **MLP / FCN**: fully connected layers.
- **CNN**: good for images because of local connections and weight sharing.
- **RNN**: good for sequence data because hidden states carry information across time.
- **Transformer**: uses attention instead of recurrence for sequence modelling.

## 2) RNN Extension: Memory Models

### What I learnt

RNNs already have a simple form of memory because the hidden state is passed from one time step to the next.

Important terms:

- **RNN**: recurrent neural network for sequence processing.
- **BPTT**: backpropagation through time.
- **LSTM / GRU**: gated RNN variants that reduce vanishing-gradient issues.
- **Memory network**: model design that adds more explicit memory.
- **Turing Machine**: abstract model of computation using a tape, read/write head, rules, and states.
- **NTM / DNC**: neural models that combine neural networks with differentiable external memory.

Key idea:

RNN-style models can be viewed as sequence processors with memory, while NTM and DNC extend this idea by giving the model a more explicit read/write memory system.

## 3) Computer Vision and Object Detection

### What I learnt

Computer vision tasks can be separated into different levels:

- **Classification**: what object is in the image?
- **Localization**: where is the object?
- **Detection**: what objects are present and where are they?
- **Segmentation**: which pixels belong to each object?

Important metric:

- **IoU** = intersection area / union area.
- A detection is usually considered correct when the class is correct and IoU is above a threshold.

Common object detection methods:

- **R-CNN / Fast R-CNN / Faster R-CNN**
  - Uses region proposals.
  - Faster R-CNN introduces the **Region Proposal Network (RPN)**.
  - Detection is split into object proposal, classification, and bounding box regression.

- **YOLO**
  - Treats detection as a regression problem.
  - Divides the image into grids.
  - Predicts bounding boxes, confidence scores, and class probabilities.
  - Faster than two-stage detectors, but may struggle with small or crowded objects.

## 5) Visual Large Models and Multimodal Learning

### What I learnt

Modern vision models increasingly use Transformer-style architectures and self-supervised learning.

Important models:

- **ViT**
  - Splits an image into patches.
  - Flattens patches into tokens.
  - Feeds image tokens into a Transformer encoder.
  - Uses a `cls` token for classification.

- **CLIP**
  - Learns from image-text pairs.
  - Uses an image encoder and text encoder.
  - Maps both image and text into the same embedding space.
  - Enables zero-shot image classification using text prompts.

- **SimCLR**
  - Uses contrastive learning for visual representation learning.
  - Learns similar representations for augmented views of the same image.

- **MAE**
  - Masks parts of an image and trains the model to reconstruct them.
  - Useful for scalable self-supervised visual learning.

## 6) LLM API Calling

### What I learnt

Calling an LLM API usually requires:

- **API_KEY**
- **BASE_URL**
- **Model ID**
- Request payload
- Response parsing

Message roles:

- `system`: sets the model behaviour and rules.
- `user`: contains the user request.
- `assistant`: stores previous model responses in the conversation.

Important ideas:

- **Single-turn calling**: one request, one response.
- **Multi-turn calling**: conversation history is passed back into the next request.
- **Context limit**: older messages may be dropped when the context is too long.
- **Context engineering**: compress or restructure important context so the model does not forget earlier information.
- **Sync vs Async**
  - Sync waits for each request to finish.
  - Async sends multiple independent requests concurrently.

## 7) Mamba and State Space Models

### What I learnt

Mamba is another approach to sequence modelling.

Comparison:

- **RNN**
  - Efficient inference.
  - Hard to parallelize training.
  - Can suffer from vanishing or exploding gradients.

- **CNN**
  - Easy to parallelize.
  - Limited context depending on kernel size.

- **Transformer**
  - Strong at long-range dependencies.
  - Training is parallelizable.
  - Attention cost grows with sequence length.

- **Mamba / SSM**
  - Based on state space models.
  - Aims for linear-time sequence modelling.
  - Uses selective scan to process long sequences efficiently.

Basic State Space Model idea:

```text
h'(t) = A h(t) + B x(t)
y(t)  = C h(t) + D x(t)
```

After discretization, the model can be computed recurrently over sequence steps. Mamba improves this by making the state update selective and efficient.

## 8) Torchvision Object Detection

### What I learnt

Torchvision is PyTorch's official computer vision library.

Main modules:

- `torchvision.datasets`: common vision datasets.
- `torchvision.transforms`: image preprocessing and augmentation.
- `torchvision.models`: pretrained model architectures.
- `torchvision.ops`: CV-specific operations such as NMS and RoIAlign.

For object detection, Torchvision provides models such as:

- Faster R-CNN
- SSD
- Mask R-CNN

A typical fine-tuning workflow:

1. Load dataset.
2. Apply transforms.
3. Load a pretrained detection model.
4. Modify the prediction head for the target classes.
5. Train and evaluate.
6. Visualize predicted boxes / masks.
