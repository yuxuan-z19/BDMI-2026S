# Study Memo - 第 8 周课程小结

**课程日期**: 2026-04-14
**课程主题**: RNN、计算机视觉任务、目标检测与多模态视觉模型

---

## 1. 课程内容概览

本周课程从序列建模继续展开，重点学习了 RNN 及其变种，同时也系统梳理了计算机视觉中的分类、检测、分割和多模态模型。

### 1.1 RNN 与序列建模
* **Vanilla RNN**: 通过隐藏状态把前一时刻的信息传递到后一时刻，适合处理文本、语音、时间序列等有顺序关系的数据。
* **BPTT**: 循环网络的训练需要通过时间反向传播展开计算图，但这也会带来梯度消失、梯度爆炸和难以并行的问题。
* **LSTM / GRU**: LSTM 通过输入门、遗忘门、输出门和记忆单元缓解长距离依赖问题；GRU 则进一步简化门结构，在参数量和训练效率上更轻量。
* **双向 RNN 与深度 RNN**: 双向结构可以同时利用前后文信息，深度堆叠则增强了模型的表达能力。

### 1.2 计算机视觉核心任务
* **图像分类**: 判断整张图像属于哪一类，是许多视觉任务的基础。
* **目标定位与检测**: 不仅要识别类别，还要给出目标所在的边界框。
* **语义分割**: 对每个像素进行分类，要求模型保留更细粒度的空间信息。
* **评价指标**: 学习了 IOU、Precision、Recall 和 mAP，这些指标比单纯的 accuracy 更适合衡量检测任务。

### 1.3 目标检测与语义分割模型
* **R-CNN 系列**: 从 R-CNN、Fast R-CNN 到 Faster R-CNN，核心变化是候选区域生成和特征复用越来越高效。
* **YOLO / SSD**: 将检测过程做成单阶段预测，速度更快，更适合实时检测场景。
* **DETR**: 使用 Transformer 做端到端检测，说明 Attention 机制也可以自然迁移到视觉任务中。
* **FCN / U-Net / DeepLab**: 语义分割从分类网络改造而来，通过上采样和多尺度特征融合恢复像素级预测。

### 1.4 多模态视觉模型
* **ViT**: 把图像切成 patch 后送入 Transformer，让图像也能以类似 token 序列的方式处理。
* **CLIP**: 通过图文对比学习把图像和文本映射到同一语义空间，实现零样本分类。
* **MAE**: 借鉴 BERT 的 mask 思想，通过遮挡图像 patch 并重构缺失部分进行自监督学习。

## 2. 个人感想

这周的内容让我对“序列模型”和“视觉模型”的联系有了更整体的认识。

1. **RNN 的局限更容易理解了**：它的结构很自然地适合序列，但顺序计算和长距离依赖问题也说明，为什么后来 Transformer 会成为主流。
2. **视觉任务不是单一的分类问题**：检测和分割对模型输出的要求更复杂，也让我意识到评价指标必须和任务目标对应起来。
3. **多模态模型体现了表示学习的重要性**：CLIP 和 MAE 让我看到，模型不只是学习标签，也可以通过图文对齐和自监督重构学到更通用的表示。
