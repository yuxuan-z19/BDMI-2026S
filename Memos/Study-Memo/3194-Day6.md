## 1.PyTorch优化模型参数
### 1.训练流程
- 前向传播：输入数据，产生输出
- 计算损失：计算损失函数
- 反向传播：计算关于参数梯度
- 参数更新：更新权重、偏置
- 迭代：重复过程

#### 迭代训练过程

本质：调整权重，最小化损失函数

梯度下降法

### 2.超参数设置
- 训练轮数（epochs，遍历次数）
- 批次大小（batch_size，更新参数前传播样本数量）
- 学习率（learning_rate，每个批次/训练轮中更新模型参数的程度）

### 3.损失函数与优化器
- 损失函数与优化器
  - nn.MSELoss
  - nn.NLLLoss
  - nn.CrossEntropyLoss
- 优化器
  - SGD
  - Adam
  - RMSprop
  - Adagrad

### 4.训练循环
1.调用`optimizer.zero_grad()`重置梯度
2.调用`loss.backword()`将预测损失反向传播
3.调用`optimizer.step()`调整参数

#### 代码示例

1.train
```python
def train_loop(dataloader, model, loss_fn, optimizer):
    size = len(dataloader.dataset)
    # Set the model to training mode - important for batch normalization and dropout layers
    # Unnecessary in this situation but added for best practices
    model.train()
    for batch, (X, y) in enumerate(dataloader):
        # Compute prediction and loss
        pred = model(X)
        loss = loss_fn(pred, y)

        # Backpropagation
        loss.backward()
        optimizer.step()
        optimizer.zero_grad()

        if batch % 100 == 0:
            loss, current = loss.item(), batch * batch_size + len(X)
            print(f"loss: {loss:.7f} [{current:>5d}]/{size:>5d}")
```

2.test
```python
def test_loop(dataloader, model, loss_fn):
    # Set the model to evaluation mode - important for batch normalization and dropout layers
    # Unnecessary in this situation but added for best practices
    model.eval()
    size = len(dataloader.dataset)
    num_batches = len(dataloader)
    test_loss, correct = 0, 0

    # Evaluating the model with torch.no_grad() ensures that no gradients are computed during test mode
    # also serves to reduce unnecessary gradient computations and memory usage for tensors with requires_grad=True
    with torch.no_grad():
        for X, y in dataloader:
            pred = model(X)
            test_loss += loss_fn(pred, y).item()
            correct += (pred.argmax(1) == y).type(torch.float).sum().item()

    test_loss /= num_batches
    correct /= size
    print(f"Test Error: \n Accuracy: {100*correct:>0.1f}%, Avg loss: {test_loss:>8f} \n")
```

## 2.多层网络结构：卷积网络
### 1.多层人工神经网络
#### 术语
- 多层神经网络 / 深度神经网络
- 前馈网络（feedforward）：MLP、CNN
- 反馈网络（feedback）：RNN、记忆网络
- 全连接（Fully Connected / Dense） vs. 稀疏连接（Sparse）

#### 层间连接关系
- 前馈：数据单向流动，无循环
- 反馈：存在反馈回路，适合序列数据
- 卷积网络属于前馈网络，采用局部连接

#### 全连接网络（MLP）
- 每个神经元与下一层所有神经元相连
- 参数量大，尤其处理图像时（例：200×200×3 的图片，单层参数可达 120k）

### 2. 卷积网络基本结构

#### 核心组件
- 卷积层（Convolutional Layer）：应用卷积核提取特征
- 激活函数层：通常使用 ReLU
- 池化层（Pooling）：下采样，减少尺寸
- 归一化层：批归一化（Batch Normalization）
- 随机丢弃层（Dropout）：防止过拟合

#### 卷积运算
- 卷积核（kernel / filter）：多维数组，参数由学习得到
- 常用卷积核数量：8、16、32、64 等
- 输出尺寸计算（有填充 P，步长 S）：
  $$
  L = \frac{W - F + 2P}{S} + 1
  $$

##### 填充与步长
- 无填充（no padding）：输出尺寸缩小
- 填充（padding）：通常填零（zero-padding）
- 步长（stride）：卷积核移动的步长

##### 多通道输入
- 输入为三维张量（高×宽×通道数），卷积核深度需与输入通道数一致
- 多个卷积核的输出堆叠形成新的张量（通道数 = 卷积核个数）

#### 池化层
- 最大池化（max-pooling）：取窗口内最大值
- 平均池化（avg-pooling）：取窗口内平均值
- 作用：减小特征图尺寸，降低参数量

#### Dropout 层
- 以概率 keep_prob 保留神经元，并乘以 1/keep_prob
- 以概率 drop_rate 输出 0
- 目的：减少过拟合

#### 卷积网络的优点
- 参数少（仅卷积核参数）
- 可并行化（不同卷积核独立运算）
- 计算快，适合硬件加速

## 3. Transformer 结构原理与代码实现

### 1. 核心术语
- 自注意力（Self‑attention）
- 可缩放的点积注意力（Scaled Dot‑Product Attention）
- 多头注意力（Multi‑head Attention）
- 位置编码（Positional Encoding）
- 遮挡（Masking）：填充遮挡（padding mask）、前瞻遮挡（look‑ahead mask）
- 点式前馈网络（Point‑wise Feed‑Forward Network）
- 编码器层 / 解码器层
- 预训练任务 / 微调任务

### 2. Transformer 整体结构
- 编码器‑解码器架构（Seq2Seq）
- 编码器：N 个相同层，每层包含：
  1. 多头自注意力（带填充遮挡）
  2. 点式前馈网络
  3. 残差连接 + 层归一化
- 解码器：N 个相同层，每层包含：
  1. 遮挡多头自注意力（前瞻遮挡 + 填充遮挡）
  2. 编码器‑解码器注意力（Q 来自解码器，K/V 来自编码器）
  3. 点式前馈网络
  4. 残差连接 + 层归一化

### 3. 注意力机制

#### 可缩放的点积注意力
- 输入：查询 Q、键 K、值 V
- 计算公式：
  $$ \text{Attention}(Q,K,V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V $$
- 缩放因子 \(\sqrt{d_k}\) 防止点积过大

#### 多头注意力
- 将 Q、K、V 线性投影到 h 个不同的低维空间（头）
- 每个头独立执行缩放点积注意力
- 拼接所有头的结果，再经过一次线性变换
- 公式：
  $$ \text{MultiHead}(Q,K,V) = \text{Concat}(\text{head}_1,\dots,\text{head}_h)W^O $$
  其中 $\text{head}_i = \text{Attention}(QW_i^Q, KW_i^K, VW_i^V)$

### 4. 位置编码
- 由于 Transformer 本身不具备顺序信息，需加入位置编码
- 使用正弦和余弦函数：
  $$ PE_{(pos,2i)} = \sin(pos / 10000^{2i/d_{\text{model}}}) $$
  $$ PE_{(pos,2i+1)} = \cos(pos / 10000^{2i/d_{\text{model}}}) $$
- 位置编码与词嵌入相加后送入编码器/解码器

### 5. 点式前馈网络
- 两层全连接网络，中间使用 ReLU 激活
- 公式：$\text{FFN}(x) = \max(0, xW_1 + b_1)W_2 + b_2$
- 对每个位置独立作用，参数共享

### 6. 代码实现要点（PyTorch）
- `ScaledDotProductAttention`：计算 QK^T，缩放，应用 mask（负无穷），softmax，加权求和
- `MultiHeadedAttention`：线性变换分头 → 并行注意力 → 拼接 → 输出线性层
- `PositionwiseFeedForward`：两个线性层 + ReLU + Dropout
- `EncoderLayer`：自注意力 + FFN，每个子层后做 Add & Norm
- `DecoderLayer`：masked 自注意力 + 编码器‑解码器注意力 + FFN
- `Encoder` / `Decoder`：堆叠 N 层 + 最终层归一化
- `make_model`：整合编码器、解码器、嵌入层、位置编码、生成器，并初始化参数

### 7. Transformer 优缺点

| 优点 | 缺点 |
|------|------|
| 不假设时间/空间关系，适合处理一组对象 | 对时序数据，每个输出都需考虑整个历史，效率较低 |
| 可并行计算（不同于 RNN 的序列计算） | 必须显式添加位置编码，否则丢失顺序 |
| 能直接建模长距离依赖，不受路径长度限制 | 计算复杂度与序列长度平方成正比 |

### 8. Vision Transformer（ViT）
- 将图像分割成固定大小的 patches（如 16×16）
- 每个 patch 线性投影为 token，加上位置编码
- 添加一个可学习的 `[CLS]` token，其最终输出用于分类
- 使用标准 Transformer 编码器
- 优势：无需卷积，可扩展到大规模图像数据集

### 9. Transformer 生命三大类

| 类别 | 代表模型 | 结构 | 主要任务 |
|------|----------|------|----------|
| 自回归（GPT 类） | GPT‑1/2/3/4，InstructGPT | Decoder‑only | 文本生成、对话、代码生成 |
| 自编码（BERT 类） | BERT，RoBERTa | Encoder‑only | 自然语言理解、分类、问答 |
| 序列到序列（T5/BART 类） | T5，BART | Encoder‑Decoder | 翻译、摘要、文本转换 |

#### BERT 要点
- 双向 Transformer 编码器
- 预训练任务：
  - 掩码语言模型（MLM）：随机 mask 15% 的 token，预测原词
  - 下一句预测（NSP）：判断两个句子是否连续
- 微调用于分类、问答、标注等任务

#### GPT 要点
- 自回归解码器，仅保留 Masked Multi‑Head Attention
- 预训练任务：标准语言模型（预测下一个 token）
- 擅长自然语言生成，通过 few‑shot / zero‑shot 适应下游任务

#### T5 / BART 要点
- T5：统一的 Text‑to‑Text 框架，所有任务都转化为文本输入 → 文本输出
- BART：结合双向编码器和自回归解码器，使用去噪目标（如 token 删除、乱序等）

### 10. 应用方向
- 聊天机器人（ChatGPT、LLaMA、GLM 等）
- 图像理解（ViT、CLIP 等多模态模型）
- 具身智能（VLA：Vision‑Language‑Action）
- 大规模预训练语言模型（DeepSeek、GPT‑4、Llama 系列）
