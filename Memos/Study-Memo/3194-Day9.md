## 1. 计算机视觉（二）：目标检测与图像分割
### 1. 评价指标
#### IoU（Intersection over Union，重叠联合比）
- 衡量预测边界框与真实边界框的重叠程度
  $$ IoU = \frac{\text{预测框与真实框的交集}}{\text{预测框与真实框的并集}} $$

#### 视觉对象检测的错误类型
| 错误类型 | 判定条件 |
|----------|----------|
| 正确（Correct） | 类别正确，IoU > 0.5 |
| 定位错误（Localization） | 类别正确，0.1 < IoU < 0.5 |
| 相似性错误（Similar） | 类别相似，IoU > 0.1 |
| 其他错误（Other） | 类别错误，IoU > 0.1 |
| 背景误认（Background） | IoU < 0.1 |

### 2. 基于深度学习的目标检测方法
#### R-CNN 系列发展
- **R-CNN** (2014)：基于区域的卷积网络，首次将深度学习用于目标检测
- **Fast R-CNN** (2015)：共享卷积计算，提高速度
- **Faster R-CNN** (2015)：引入 RPN（区域生成网络），端到端训练

#### Faster R-CNN
- **结构**：RPN + Fast R-CNN
- **RPN 工作原理**：在特征图上滑动 3×3 窗口，每个位置生成 9 个 anchor box（3 种面积 × 3 种比例）
- **4 个损失函数**：
  1. RPN 分类（是否为目标）
  2. RPN 边界框回归
  3. Fast R-CNN 分类
  4. Fast R-CNN 边界框回归
- **效果**：ResNet-101 版本速度是 R-CNN 的 250 倍，Fast R-CNN 的 10 倍
- **训练方式**：轮流训练、近似联合训练、联合训练

### 3. YOLO 系列（You Only Look Once）
#### 核心思想
- 将目标检测视为**回归问题**，直接从图像预测边界框和类别
- 速度极快：Titan X GPU 上可达 45 FPS（实时）

#### YOLO v1 流程
1. 将图像分割为 S×S 网格，每个网格负责检测中心落在其中的目标
2. 每个网格预测 B 个边界框，每个框包含 (x, y, w, h, confidence)
3. 置信度 = P(Object) × IoU

#### 网络结构
- 24 个卷积层 + 2 个全连接层，借鉴 GoogleNet 和 NIN
- 输出：S × S × (B × 5 + #Classes) 的三维张量

#### 缺点
- 对长宽比特殊的物体、成群的小目标容易漏检
- 精度不如 Faster R-CNN

#### YOLO 版本演进
YOLOv1 → v2/v3 → v4 → v5 → v6 → v7 → v8 → v9 → v10 → v11

### 4. SSD（Single Shot MultiBox Detector）
#### 核心特点
- 在不同层次的特征图上预测不同尺度的目标
- 使用小卷积核预测边界框偏移和类别得分
- 每个特征图格子预定义一系列 **default box**（不同纵横比）

#### 后处理
- 使用**非极大值抑制（NMS）** 得到最终预测

### 5. DETR（DEtection TRansformer）
- 基于 Transformer 的端到端目标检测方法
- 无需 anchor、NMS 等手工设计组件
- 结构：CNN 骨干网络 + Transformer 编码器‑解码器
- 解码器接收 object queries，输出类别标签和边界框

### 6. 图像语义分割
#### 定义
- 对图像中的**每一个像素**进行分类
- 输出与输入大小相同的分割图

#### FCN（全卷积网络，2014）
- **里程碑**：将图像分类扩展到像素级分类
- **结构**：卷积+池化（下采样）→ 上采样恢复原始分辨率
- **跳跃连接**：融合浅层（细节丰富）与深层（语义强）特征图，如 FCN-16s
- **缺点**：像素独立分类，未考虑像素间关系；细节不够精细

#### 后续发展
| 模型 | 特点 |
|------|------|
| SegNet (2015) | Encoder-Decoder 对称结构，上池化层，边缘刻画好 |
| UNet (2015) | U 型结构，特征图相叠（非相加），多尺度融合，适合医学图像 |
| PSPNet (2017) | 金字塔池化，融合多尺度上下文信息 |
| DeepLabv3 | 空洞卷积（atrous convolution）+ 可分离卷积 |

#### 工具库
- Keras2_segmentation：包含 FCN、SegNet、UNet、PSPNet 等预训练模型

### 7. Transformer 在分割中的应用
#### 动机
- CNN 擅长局部特征，但忽视长期依赖和全局上下文

#### 代表性方法
| 模型 | 特点 |
|------|------|
| TransUNet | Transformer + CNN 编码器，用于医学图像分割 |
| Swin Transformer | 分层特征图 + 移位窗口，高效处理高分辨率图像 |
| Swin-UNet | 纯 Transformer 架构（无 CNN），U 形结构 |
| UNETR | Transformer 编码器 + CNN 解码器，用于 3D 医学图像 |
| HiFormer | CNN + Transformer 混合特征提取 |
| TransBTS | Transformer 增强低分辨率阶段的分割性能 |

## 2. 计算机视觉（三）：自动驾驶应用
### 1. 基本概念
- **自主/无人驾驶（Self-Driving / Driverless）**：通过控制车辆速度、方向和刹车，直接接替人类驾驶员控制车辆。
- **辅助驾驶（ADAS）**：利用激光雷达、相机、GPS 等传感器感知环境，通过决策算法提醒驾驶员注意道路状况。

### 2. 自动驾驶的传感器与数据
| 传感器类型 | 作用 |
|------------|------|
| 激光雷达 | 三维环境建模 |
| 摄像头 | 视觉识别（交通标志、行人、车辆等） |
| 声纳雷达 | 近距离障碍物检测 |
| 北斗/GPS | 定位与导航 |

### 3. 自动驾驶中的 AI 应用
- 交通标志检测（信号灯、限速牌等）
- 行人检测（Pedestrian Detection）
- 车辆检测（卡车、自行车、轿车等）
- 车道线识别（转弯线、车道保持）
- 驾驶员人脸识别（疲劳监测等）

### 4. 自动驾驶原理（以 NVIDIA 为例）
#### 驾驶感知模块
- **LaneNet**：预测车道线
- **PathNet**：预测可运行路径的边线（无论有无车道线）
- **PilotNet**：根据人类驾驶轨迹预测中心路径

#### NVIDIA DRIVE 平台
- 提供感知、规划、控制等完整开发方案

### 5. 特斯拉自动驾驶（Tesla FSD）
- **纯视觉方案**：HydraNets 神经网络
- 输入：8 个摄像头，1280×960 12‑Bit HDR，36Hz
- 处理：单个神经网络整合为 **3D 环境感知**（Vector Space）
- 特点：基于深度卷积网络，具有**多任务学习能力**

## 3. Torchvision 目标检测
### 1. Torchvision
- **定义**：PyTorch 官方为计算机视觉任务打造的扩展库
- **功能**：包含预训练视觉模型、数据集接口、数据增强工具、CV 专用算子
- **优势**：避免重复造轮子，与 PyTorch 无缝衔接

### 2. 核心模块
| 模块 | 说明 |
|------|------|
| `torchvision.datasets` | 内置经典数据集（ImageNet、CIFAR‑10、COCO 等）的下载与加载接口 |
| `torchvision.transforms` | 图像预处理与数据增强（随机裁剪、旋转、颜色变换、转张量等） |
| `torchvision.models` | 经典网络架构及预训练权重（分类、检测、分割模型） |
| `torchvision.ops` | CV 专用高效算子（非极大值抑制 NMS、RoIAlign 等） |

### 3. 目标检测模型
- **Faster R‑CNN**：两阶段检测器，高精度（arXiv:1703.06870）
- **SSD**：单阶段检测器，速度快（arXiv:1512.02325）
- **Mask R‑CNN**：在检测基础上增加实例分割（预测 mask）

#### 自定义数据集与微调
- 自定义数据集需继承 `torch.utils.data.Dataset`，实现 `__len__` 和 `__getitem__` 方法。
- 每个样本返回 `(image, target)`，其中 `target` 为字典，包含：
  - `boxes`：形状 `(N, 4)` 的浮点张量（`[x1, y1, x2, y2]`）
  - `labels`：形状 `(N,)` 的整数张量（类别标签，背景为 0）
  - `image_id`：图像标识（整数）
  - `area`：边界框面积（可选，用于评估）
  - `iscrowd`：是否为密集目标（可选，通常设为 0）

```python
class CustomDataset(torch.utils.data.Dataset):
    def __init__(self, root, transforms=None):
        self.root = root
        self.transforms = transforms
        self.imgs = list(sorted(os.listdir(os.path.join(root, "images"))))
        self.annotations = ...  # 加载标注文件

    def __getitem__(self, idx):
        img_path = os.path.join(self.root, "images", self.imgs[idx])
        img = Image.open(img_path).convert("RGB")
        boxes, labels = self.load_annotations(idx)  # 自定义解析函数
        target = {"boxes": boxes, "labels": labels}
        if self.transforms:
            img, target = self.transforms(img, target)
        return img, target
```

#### 微调预训练模型
```python
from torchvision.models.detection import fasterrcnn_resnet50_fpn

model = fasterrcnn_resnet50_fpn(pretrained=True)
num_classes = 21  # 20个自定义类 + 背景
in_features = model.roi_heads.box_predictor.cls_score.in_features
model.roi_heads.box_predictor = torchvision.models.detection.faster_rcnn.FastRCNNPredictor(in_features, num_classes)
```

#### 加入 Mask R‑CNN
```python
from torchvision.models.detection import maskrcnn_resnet50_fpn
model = maskrcnn_resnet50_fpn(pretrained=True)

# 修改 mask 预测头
in_features_mask = model.roi_heads.mask_predictor.conv5_mask.in_channels
hidden_layer = 256
model.roi_heads.mask_predictor = torchvision.models.detection.mask_rcnn.MaskRCNNPredictor(in_features_mask, hidden_layer, num_classes)
```

#### 训练与测试
- 定义优化器（SGD 或 Adam）
- 使用 DataLoader 批量加载数据（需自定义 collate_fn）
- 训练循环：前向传播 → 计算损失 → 反向传播 → 优化器 step
- 测试时设置 `model.eval()`，使用 `torch.no_grad()`，对输出应用置信度阈值和 NMS。
```python
for images, targets in dataloader:
    images = [img.to(device) for img in images]
    targets = [{k: v.to(device) for k, v in t.items()} for t in targets]
    loss_dict = model(images, targets)
    losses = sum(loss for loss in loss_dict.values())
    optimizer.zero_grad()
    losses.backward()
    optimizer.step()
```
