## 循环网络
### 基本循环网络（Vanilla RNN）
- 循环网络通过反馈连接将当前状态传递到下一步计算
- 不同时间步共享相同权重矩阵（U：输入→隐藏，V：隐藏→输出，W：隐藏→隐藏）
- 在时间维度展开计算图，通过时间反向传播（BPTT）训练
- 特点：权重共享、处理序列数据，但训练代价高、无法并行化
- BPTT 的梯度消失/爆炸问题：常使用梯度截断（Gradient Clipping）加以控制
- 扩展结构：深度 RNN（堆叠多层）、双向 RNN（利用过去和未来信息）

### LSTM（长短时记忆网络）
- 引入记忆单元（Cell）+ 三个门：输入门、遗忘门、输出门
- 遗忘门控制是否存储历史信息，输入门控制是否输入，输出门控制是否输出
- Cell 作为积分器对时间累加，通过额外路径传播梯度，缓解梯度消失
- 解决了 Vanilla RNN 收敛慢和长距离依赖问题

### GRU（门控循环单元）
- LSTM 的简化版（Cho et al., 2014），参数更少、运算更快
- 合并记忆状态和隐藏状态为一个状态
- 合并输入门和遗忘门为一个更新门（Update Gate），再加一个重置门（Reset Gate）
- 与 LSTM 性能相当，但计算成本更低


## 计算机视觉（Computer Vision）
### 传统方法
- OpenCV：图像处理库（pip install opencv-python），支持颜色识别、边缘检测等
- scikit-image：颜色空间转换（RGB-HSV）、图像分割（分水岭算法）
- 局部特征点检测：SIFT、SURF、Harris 角点、FAST
- 传统机器学习：SVM、朴素贝叶斯、决策树、HOG、Boosting、Haar 分类器（人脸）

### 卷积网络（CNN）
- AlexNet (2012)：8 层、6 千万参数、GPU 训练，带动深度学习在 CV 的爆发
- 发展路线：AlexNet → NiN → VGG → GoogLeNet → ResNet → DenseNet
- ResNet 引入残差连接（Skip Connection）解决深层网络退化问题

### Vision Transformer（ViT）
- 将图像切分成固定大小的 Patches，线性嵌入后加位置编码，送入标准 Transformer 编码器
- 变体：DeiT（蒸馏训练）、Swin Transformer（分层+移位窗口）、LeViT、XciT（交叉协方差注意力）
- MLP-Mixer：纯 MLP 架构，无需卷积和注意力

### CNN / ViT / MLP-Mixer 对比
| 特性 | CNN | ViT | MLP-Mixer |
|------|-----|-----|-----------|
| 核心算子 | 卷积（局部） | 自注意力（全局） | 纯 MLP |
| 感受野 | 逐层扩大 | 直接全局 | 直接全局 |
| 计算复杂度 | O(S) | O(S²) | O(S) |
| 位置信息 | 天然自带 | 需位置编码 | 无需显式编码 |
| 小数据集 | 好 | 一般 | 较差 |

### AbSViT（综合式分析视觉 Transformer）
- 基于贝叶斯推断的 "Analysis by Synthesis"，视觉感知同时受输入图像和先验（prior）指导
- 稀疏重建与视觉注意力功能等价，每一层计算受 top-down signal 调控的视觉注意力
- 可通过自然语言控制 attention，在 VL 任务上表现优越

### 评价指标
- 精确率（Precision）= TP/(TP+FP)
- 召回率（Recall）= TP/(TP+FN)
- 准确率（Accuracy）= (TP+TN)/总数
- IOU（交并比）：衡量预测框与真实框的重叠程度
- mAP（Mean Average Precision）：平衡精确率与召回率，目标检测核心指标
- PR 曲线 AUC：面积越大，检测效果越好
- F1 分数：精确率和召回率的调和平均

### 对象检测方法
- R-CNN 系列：
  - R-CNN：选择性搜索提取候选区域 + CNN 分类
  - Fast R-CNN：RoI Pooling 加速
  - Faster R-CNN：RPN（区域生成网络）替代选择性搜索，anchors 机制（3 种尺度 × 3 种比例 = 9 个 anchor），速度提升 250 倍
- YOLO（You Only Look Once）：将检测转化为回归问题，单次前向传播预测边界框和类别，速度快（45 FPS）
- SSD（Single Shot MultiBox Detector）：多尺度特征图上预测，兼顾速度与精度
- DETR：基于 Transformer 的端到端目标检测，CNN backbone + Transformer encoder-decoder

### 语义分割
- 目标：对图像每个像素进行分类
- FCN（全卷积网络，2014）：用卷积替代全连接 + 上采样恢复分辨率，里程碑意义
- SegNet：Encoder-Decoder 对称结构 + 上池化
- UNet：U 型结构，多尺度特征图融合（拼接而非相加），擅长大图像
- PSPNet：金字塔池化，多尺度上下文融合
- DeepLabv3：空洞卷积，多尺度特征提取
- Mask R-CNN：实例分割
- 基于 Transformer 的分割：TransUNet、Swin-UNet、UNETR（3D 医学图像）、HiFormer（CNN+Transformer 混合）、TransBTS

## 多模态视觉大模型
### 主要模型
- ViT（Google）：将 Transformer 应用于图像分类，"An Image is Worth 16x16 Words"
- CLIP（OpenAI）：图像编码器 + 文本编码器映射到共同隐空间，对比学习，支持零样本分类
- MAE（MetaAI）：遮掩自编码器，随机遮掩 75% 图像块后重构，非对称编码器-解码器设计，自监督学习
- CogVLM：国内视觉大模型
- GLM4v-9B：智谱多模态模型

### CLIP 原理
- 图像编码器（CNN/ResNet）提取视觉特征
- 文本编码器（Transformer/BERT 变体）提取语义特征
- 对比学习预训练：拉近匹配的图文对的嵌入，推远不匹配的
- 三阶段：对比预训练 → 提取类别文本特征 → 零样本推理

### MAE 核心设计
- 非对称编码器-解码器：编码器仅处理可见 patches（25%），解码器重建全部 patches
- 高遮掩率（75%）迫使模型学习全局信息
- 解码器更轻量（计算量仅编码器 10%）
- 预训练后在检测、分割等下游任务上表现优异

## Transformer 注意力机制发明史
### 语言模型发展脉络
- 统计语言模型：预测文档中下一个词的概率
- Yoshua Bengio (2003)：用神经网络学习语言模型
- Alex Graves (2013)：用 LSTM 生成具有长距离结构的复杂序列
- Seq2Seq (Ilya Sutskever, 2014)：Encoder-Decoder 架构用于机器翻译

### 注意力机制的诞生
- Bahdanau 等 (2015, ICLR)：首次在 NMT 中引入注意力机制，让模型在翻译时动态关注输入的相应部分，解决了长句翻译的瓶颈
- 后续推动：记忆网络（Sukhbaatar, 2015）、内部注意力（Cheng, 2016）、自注意力句子嵌入（Lin, 2017）
- Transformer (Vaswani et al., 2017, "Attention is All You Need")：完全基于注意力和全连接，去掉了 RNN 序列结构，支持并行训练

## Transformer 推理效率优化
### KV Cache 原理
- 推理时预填充阶段计算 K、V 并缓存，后续解码时直接读取，仅计算新 Query
- 计算复杂度从 O(n²) 降至 O(n)，大幅提升生成效率
- 两个阶段：Prefill（预填充）→ Decode（逐 token 生成）

### 注意力优化策略
- MQA（Multi-Query Attention）：多个头共享 K、V 减少内存
- MLA（Multi-head Latent Attention, DeepSeek）：低秩键值联合压缩，消除 KV 缓存瓶颈
- Sparse Transformer：稀疏注意力降低计算量
- Sliding Window Attention：仅关注最近 N 个 token
- Flash Attention：高效注意力实现
- Attention Sink 现象：前几个 token 吸收了过多注意力

### 自适应 KV 缓存管理策略
- FastGen：根据注意力头特性定制保留策略
- H2O（Heavy Hitter Oracle）：动态驱逐，平衡近期 token 和历史重要 token
- SnapKV：基于注意力分数选择/聚类重要 KV 位置
- PyramidKV / PyramidInfer：金字塔 KV 缓存压缩
- StreamingLLM：使有限窗口模型处理无限长度序列
- LM-Infinite：2K-4K 预训练模型泛化到 200M 长度

### 推理引擎
- vLLM：高性能 LLM 推理引擎，创新内存管理（PagedAttention），支持高并发
- SGLang：后端运行时 + 前端语言协同设计，高效处理结构化 LM 程序

### 其他推理加速技术
- 模型压缩：剪枝、量化、条件计算（CoLT5）
- Reformer：局部敏感哈希降低注意力计算复杂度
- Performer：正正交随机特征近似注意力核
- 语义缓存：基于语义相似度的提示缓存，可降低推理延迟达 100 倍（VectorQ）


## Transformer 推理的硬件加速
### 浮点数精度
- 单精度 FP32、半精度 FP16、BF16、FP8 等
- AI 应用中 6 种浮点数表示，低精度加速推理的同时需平衡精度损失

### Intel CPU 加速
- AVX（Advanced Vector Extensions）：SIMD 指令集，AVX-512 一次处理 512 位数据
- AMX（Advanced Matrix Extensions）：Sapphire Rapids 引入，专为矩阵乘法设计的硬件加速单元
  - 二维 tile 寄存器（16 行 × 512 位），共 8 个
  - 支持 BF16 和 INT8 格式
- LLM 中线性层和注意力机制的核心计算均为矩阵乘法（GEMM），AMX 直接加速
- 稀疏权重解压缩处理进一步优化

### FPGA
- 可编程硬件加速方案


## Torchvision 目标检测
### Torchvision 四大核心模块
- datasets：内置 ImageNet、CIFAR-10、COCO 等经典数据集接口
- transforms：图像预处理与数据增强（裁剪、旋转、颜色变换等）
- models：预训练模型库
- ops：视觉专用算子（NMS、RoIAlign 等）

### 目标检测模型
- Faster R-CNN：两阶段检测器
- SSD：单次检测器
- Mask R-CNN：同时预测检测框和分割 mask

### 微调流程
- 加载预训练权重 → 修改模型头部适配类别数 → 定义数据集和数据增强 → 训练和评估
