# Week 9 Study Memo — Deep Learning Overview, RNN Extensions & Multimodal AI

## 1. Deep Learning Overview (Comprehensive Review)

### From Linear Classifiers to Deep Networks
1. **Linear Classifier (SVM)**: Image classification via linear decision boundaries
   - Loss function → Optimization → Backpropagation
   - Limitation: Cannot handle nonlinear problems
2. **Deep Neural Networks**: Stack nonlinear layers to learn complex patterns

### Training Deep Neural Networks
| Step | Details |
|------|---------|
| Data preprocessing | Normalization, augmentation |
| Network design | Choose architecture + loss function |
| Gradient descent | SGD, Adam, RMSProp |
| Learning rate | Exponential decay, 1/t decay, step decay |
| Regularization | L2, Dropout, Data augmentation, Ensemble learning |

### Overfitting Solutions
- **L2 regularization**: Penalize large weights
- **Data augmentation**: Flip, rotate, scale, crop, add noise
- **Dropout**: Randomly zero out neurons during training
- **Ensemble learning**: Combine multiple models

## 2. RNN Extensions

### RNN Recap
- Vanilla RNN → LSTM → GRU (covered in previous weeks)
- BPTT (Back-Propagation Through Time) for training

### Advanced RNN Architectures
- **Bidirectional RNN**: Process sequences in both forward and backward directions
- **Deep RNN**: Stack multiple RNN layers
- **Differentiable Neural Computer (DNC)**: RNN with external memory (DeepMind)

### Sequence-to-Sequence (Seq2Seq)
- Encoder maps input sequence to a fixed-size context vector
- Decoder generates output sequence from the context vector
- Applications: machine translation, text summarization

## 3. Transformer (Continued)

### Architecture Review
- Encoder-decoder with self-attention and cross-attention
- Scaled dot-product attention: $\text{Attention}(Q,K,V) = \text{softmax}(QK^T/\sqrt{d_k})V$
- Multi-head attention for parallel feature learning
- Positional encoding via sine/cosine functions

### Transformer Life Tree — Three Families
| Family | Model Examples | Type |
|--------|---------------|------|
| **Autoregressive** | GPT, GPT-2, GPT-3, GPT-4 | Decoder-only |
| **Autoencoding** | BERT, RoBERTa | Encoder-only |
| **Seq2Seq** | T5, BART | Encoder-decoder |

### Key Paper
- *"Attention Is All You Need"* (Vaswani et al., 2017) — foundational Transformer paper.

## 5. Mamba & State Space Models (SSM)

### Motivation: Limitations of Existing Sequence Models
| Model | Training | Inference | Context |
|-------|----------|-----------|--------|
| **RNN** | $O(N)$, not parallelizable | $O(1)$ per token | Theoretically infinite, but vanishing gradients |
| **CNN** | Parallelizable | Depends on kernel size | Finite (kernel-limited) |
| **Transformer** | $O(N^2)$, parallelizable | $O(N)$ per token (with KV cache) | Finite (context window) |

**Ideal model**: Parallelizable training like Transformer + $O(1)$ inference like RNN → **State Space Models**.

### State Space Model (SSM) Basics
- Maps input signal $x(t)$ to output $y(t)$ via a latent state $h(t)$:
  - State equation: $h'(t) = Ah(t) + Bx(t)$
  - Output equation: $y(t) = Ch(t) + Dx(t)$
- Parameters A, B, C, D are **fixed** (time-invariant), making it a **Linear Time-Invariant (LTI)** system.

### Discretization
- Continuous SSM → discrete recurrence via Euler's method or Zero-Order Hold (ZOH):
  - $h_{k+1} = \bar{A}h_k + \bar{B}x_k$ where $\bar{A} = I + \Delta A$, $\bar{B} = \Delta B$
- Step size $\Delta$ is a **learnable parameter**.

### Dual Computation Modes
| Mode | Use Case | Advantage |
|------|----------|----------|
| **Recurrent** | Inference | $O(1)$ per token, constant memory |
| **Convolutional** | Training | Parallelizable, no sequential dependency |

### HIPPO Matrix
- Careful initialization of the **A matrix** using HIPPO theory.
- Approximates input history using Legendre polynomials.
- Recent inputs are remembered precisely; older ones decay exponentially.

### Mamba: Selective State Space Model
- **Key innovation**: Makes B, C, and $\Delta$ **input-dependent** (time-varying), enabling **content-aware reasoning**.
- Unlike vanilla SSM, Mamba can "select" which information to remember/forget based on input content.
- Cannot use convolutional mode (parameters differ per token) → uses **parallel scan** algorithm instead.

### GPU Optimization Techniques
- **Kernel fusion**: Combine multiple CUDA operations to avoid HBM↔SRAM data transfers.
- **Activation recomputation**: Recompute activations during backprop instead of caching them.
- **Parallel scan**: Parallelize recurrent computation using associative property ($O(N/T)$ with $T$ threads).

### Mamba Architecture
- Built by stacking **Mamba blocks** (similar to stacking Transformer layers).
- Each block: Linear projection → Conv1D → SSM → SiLU activation → Output projection.
- Derived from the H3 (Hungry Hungry Hippos) architecture.
