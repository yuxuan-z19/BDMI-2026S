## 1) Computer Vision Basics

### What I learnt

Computer vision is about letting computers process and understand images or videos.

Common visual tasks:

- **Image classification**: predict the object class in an image
- **Localization**: predict where the object is
- **Object detection**: predict both class and bounding box
- **Semantic / instance segmentation**: label pixels or object masks

A useful way to remember:

```text
Classification  -> What is in the image?
Localization    -> Where is the main object?
Detection       -> What objects are there and where?
Segmentation    -> Which pixels belong to each object/class?
```

## 2) Human Vision as Reference

### What I learnt

The lesson first used human perception as a reference for computer vision.

Important ideas:

- Humans receive a large amount of information through vision.
- The eye performs early visual processing through the retina.
- Rod cells are mainly for brightness / low-light vision.
- Cone cells are mainly for colour vision.
- The brain processes visual signals through different visual pathways and cortex areas.

This gives inspiration for deep learning vision systems: process raw visual signals step by step, from low-level patterns to high-level understanding.

## 3) CNN for Image Processing

### What I learnt

A CNN uses convolution layers to extract visual features.

Key ideas:

- A **kernel/filter** slides over the image and performs weighted sums.
- CNNs use **local connections**, unlike fully connected layers.
- The same filter is reused across the image, so CNNs use **parameter sharing**.
- Pooling helps reduce spatial size and learn more stable features.

Typical CNN block:

```text
Convolution -> Activation -> Pooling / Normalization -> Dropout
```

Why CNNs are useful:

- Fewer parameters than fully connected networks
- Can exploit local image structure
- Faster and easier to parallelize
- Good for image classification and detection

## 4) Object Detection Metrics

### What I learnt

Object detection is evaluated using both classification and localization quality.

Important metrics:

- **IOU (Intersection over Union)** measures overlap between predicted box and ground-truth box.
- **Precision** asks: among predicted positives, how many are correct?
- **Recall** asks: among actual positives, how many are found?
- **Accuracy** is the overall proportion of correct predictions.

Detection error types:

- **Correct**: correct class and IOU > 0.5
- **Localization error**: correct class but weak overlap
- **Similar error**: similar class with overlap
- **Background error**: predicted object where there is no object

## 5) R-CNN Family and Faster R-CNN

### What I learnt

The R-CNN family introduced region-based detection.

Progression:

```text
R-CNN -> Fast R-CNN -> Faster R-CNN
```

Faster R-CNN has two main parts:

1. **RPN (Region Proposal Network)**
   - Proposes candidate object regions.
   - Uses anchors with different scales and ratios.

2. **Fast R-CNN head**
   - Classifies each proposal.
   - Refines the bounding box.

Faster R-CNN improves speed by replacing selective search with an RPN.

## 6) YOLO and Single-stage Detection

### What I learnt

YOLO treats object detection as a direct regression problem.

Main idea:

- Divide the image into grids.
- Each grid predicts bounding boxes, confidence scores, and class probabilities.
- Apply confidence thresholding and **NMS (Non-Maximum Suppression)** to remove duplicate boxes.

Strengths:

- Very fast
- Suitable for real-time detection
- Simple end-to-end pipeline

Limitations:

- Early YOLO versions struggled with small or crowded objects.
- Accuracy may be lower than two-stage detectors such as Faster R-CNN.

## 8) Vision Transformers and Multimodal Models

### What I learnt

Transformers can also be used for vision tasks.

### ViT

Vision Transformer (ViT) splits an image into patches, flattens each patch, and treats the patches like tokens in a Transformer.

Basic flow:

```text
Image -> Patches -> Patch embeddings -> Transformer encoder -> Classifier
```

### CLIP

CLIP learns a shared space between images and text.

It has:

- An image encoder
- A text encoder
- Contrastive learning using image-text pairs

This allows zero-shot prediction using text labels.

### MAE

Masked Autoencoder (MAE) learns visual representations by masking image patches and reconstructing them.

## 9) Torchvision Object Detection

### What I learnt

Torchvision provides ready-made tools for computer vision.

Main modules:

- `torchvision.datasets`: common datasets
- `torchvision.transforms`: preprocessing and augmentation
- `torchvision.models`: pretrained vision models
- `torchvision.ops`: CV-specific operations such as NMS and RoIAlign

Example workflow:

```python
import torchvision

model = torchvision.models.detection.fasterrcnn_resnet50_fpn(weights="DEFAULT")
model.eval()

# Use a transformed image tensor as input:
# output = model([image_tensor])
```

For fine-tuning, the usual steps are:

1. Prepare dataset and labels.
2. Apply image transforms.
3. Load a pretrained detection model.
4. Replace the prediction head for the target classes.
5. Train and evaluate using detection metrics.
