# Week 3 Study Memo — Machine Learning Fundamentals & Scikit-learn

## 1. Machine Learning Overview

### Definition: The ⟨T, E, P⟩ Framework
- **T** (Task): A computational task (e.g., spam classification, face recognition)
- **E** (Experience): Training data / dataset
- **P** (Performance): A measurable metric that improves with experience

> Machine learning uses experience set E to improve the performance P on task T.
> — Prof. Tom M. Mitchell, CMU

### Why Machine Learning?
- **Adaptability**: ML models automatically adapt to changing data (e.g., evolving spam patterns)
- **Data Mining**: Discovers non-obvious patterns in large datasets that humans can't manually inspect

### ML Applications Discussed
- Spam filtering, face recognition (Megvii/Face++), credit assessment, fraud detection, weather prediction, recommendation systems

---

## 2. Types of Machine Learning

### By Supervision Level
| Type | Description | Example |
|------|-------------|---------|
| **Supervised** | Labeled data with features + labels | Classification, regression |
| **Unsupervised** | No labels; discover structure | Clustering, anomaly detection |
| **Semi-supervised** | Mix of labeled and unlabeled data | Image classification with few labels |
| **Reinforcement** | Agent learns via rewards/penalties | Game playing, robotics |

### By Learning Mode
- **Online learning**: Incremental, processes data in streams
- **Batch learning**: Learns from entire dataset at once

### By Approach
- **Model-based**: Build a model from training data, use it for predictions
- **Instance-based**: Compare new instances to learned examples via similarity measures (e.g., recommender systems)

### ML Challenges
- Insufficient training data
- Non-representative training data
- Low-quality data (errors, outliers, noise)
- Irrelevant features

### Datasets
- **Training set**: Used to train the model
- **Validation set**: Subset of training set for tuning
- **Test set**: Final evaluation of model performance

---

## 3. ML vs. Related Fields
- **Statistics**: Hypothesis → Data → Test (top-down)
- **Machine Learning**: Data → Models → Patterns (bottom-up)
- **Data Science**: Using data to understand and solve real-world problems

## 4. Scikit-learn Practice

### Overview
- Scikit-learn is a Python ML framework built on NumPy/SciPy.
- Import pattern: `from sklearn.XXX import YYY`

### Supervised Learning Methods Practiced
| Method | Abbreviation | Use Case |
|--------|-------------|----------|
| Gaussian Naïve Bayes | GNB | Text classification |
| Decision Trees | DT | Interpretable classification |
| Logistic Regression | LR | Binary/multi-class classification |
| Support Vector Machine | SVM | Complex decision boundaries |

### Unsupervised Learning Methods
| Method | Use Case |
|--------|----------|
| K-Means Clustering | Grouping similar data points |
| Gaussian Mixture Model (GMM) | Soft clustering with probability |

### Data Preprocessing
- Feature transformation & scaling
- Train/test split: `from sklearn.model_selection import train_test_split`

### Pi Calculation (Monte Carlo Simulation)
- Application of random number generation to estimate π
- Based on ratio of random points falling inside a unit circle vs. unit square
